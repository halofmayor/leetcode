# 📝 Append Characters to String to Make Subsequence

| Difficulty | Topics |
| :--- | :--- |
| Medium | Two Pointers, String |

## 📖 Problem Description
You are given two strings `s` and `t`. Return the minimum number of characters that need to be appended to the end of `s` so that `t` becomes a subsequence of `s`.

---

## 💡 Initial Thoughts & Evolution
Since I solved "Is Subsequence" recently, I realized the logic is nearly identical. Instead of checking if the entire string `t` is already there, I just need to find the longest prefix of `t` that is already a subsequence of `s`. Anything left over in `t` must be appended.

> Key Insight: The number of characters to append is simply the total length of `t` minus the number of characters we successfully matched in order.

---

## 🚀 Final Solution

    class Solution {
        public int appendCharacters(String s, String t) {
            // If t is empty, it's already a subsequence
            if (t.length() == 0) {
                return 0;
            }

            char[] target = t.toCharArray();
            int count = 0;

            for (int i = 0; i < s.length(); i++) {
                // Check if current char in 's' matches the next needed char in 't'
                if (target[count] == s.charAt(i)) {
                    count++;
                    
                    // If we matched all of 't', no characters need to be appended
                    if (count == t.length()) {
                        return 0;
                    }
                }
            }
            
            // The remaining characters in 't' are the ones we need to append
            return t.length() - count;
        }
    }

---

## 🛠️ Algorithm
1. Early Exit: If `t` is empty, return 0.
2. Conversion: Convert `t` to a `char[]` for indexed access.
3. Matching Loop: Iterate through `s` once.
4. Greedy Match: Every time `s.charAt(i)` matches the current character we are looking for in `t` (`target[count]`), increment `count`.
5. Early Finish: If `count` reaches the length of `t` during the loop, return 0 immediately.
6. Calculation: After the loop, the characters from `count` to the end of `t` were not found. Return `t.length() - count`.

---

## 📊 Complexity Analysis
* Time Complexity: O(n) — Where n is the length of string `s`. We scan `s` exactly once.
* Space Complexity: O(m) — Where m is the length of string `t`, due to `t.toCharArray()`.

## 📝 Code Notes
* Greedy Approach: We match characters as early as possible in `s` to maximize the chances of completing the subsequence.
* Return Logic: `t.length() - count` correctly handles cases where 0 characters were matched or all but one were matched.
