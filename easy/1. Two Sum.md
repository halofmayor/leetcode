# 📝 Two Sum

| Difficulty | Topics |
| :--- | :--- |
| Easy | Array, Hash Table |

## 📖 Problem Description
Given an array of integers `nums` and an integer `target`, return indices of the two numbers such that they add up to `target`.

You may assume that each input would have **exactly one solution**, and you may not use the same element twice.

---

## 💡 Initial Thoughts & Evolution
My first approach was the "Brute Force" method: using nested loops to check every possible pair. While it works ($O(n^2)$), it's inefficient for large datasets. I realized that for any number `x`, I am specifically looking for a `complement` (where `complement = target - x`).

> Key Insight: Instead of searching for the complement by looping again, we can store the numbers we've already seen in a HashMap for $O(1)$ instant lookup.

### ❌ Initial Attempt (Brute Force)
    class Solution {
        public int[] twoSum(int[] nums, int target) {
            for (int i = 0; i < nums.length; i++) {
                for (int j = i + 1; j < nums.length; j++) {
                    if (nums[i] + nums[j] == target) {
                        return new int[] { i, j };
                    }
                }
            }
            return null;
        }
    }

---

## 🚀 Final Solution

    class Solution {
        public int[] twoSum(int[] nums, int target) {
            // Map to store: Key = Number Value, Value = Index
            Map<Integer, Integer> numMap = new HashMap<>();
            
            for (int i = 0; i < nums.length; i++) {
                int complement = target - nums[i];
                
                // Check if the number we need exists in our "history"
                if (numMap.containsKey(complement)) {
                    return new int[]{numMap.get(complement), i};
                }
                
                // Store the current number and its index for future pairs
                numMap.put(nums[i], i);
            }

            return new int[]{}; 
        }
    }   

---

## 🛠️ Algorithm
1. Initialization: Create a `HashMap` to store values as keys and their array indices as values.
2. Single Pass: Iterate through the array once.
3. Math: For the current element, calculate the `complement` needed to reach the `target`.
4. Lookup: Check if that `complement` is already in the map.
5. Success: If found, return the index stored in the map and the current index.
6. Record: If not found, add the current number and its index to the map and move to the next.

---

## 📊 Complexity Analysis
* Time Complexity: O(n) — We traverse the array only once. Each lookup in the HashMap takes $O(1)$ on average.
* Space Complexity: O(n) — In the worst case, we store almost every element of the array in the map.

## 📝 Code Notes
* One-Pass Optimization: We don't need to fill the map first. By checking and adding in the same loop, we ensure we don't use the same element twice and keep the logic lean.
* Complement Logic: `target - nums[i]` is the "missing piece" we are hunting for.
