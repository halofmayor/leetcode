# 📝 Replace Elements with Greatest Element on Right Side

| Difficulty | Topics |
| :--- | :--- |
| Easy | Array, Iteration |

## 📖 Problem Description
Given an array `arr`, replace every element in that array with the greatest element among the elements to its right, and replace the last element with `-1`.

---

## 💡 Initial Thoughts & Evolution
My first instinct was to loop through each element and then run another loop to find the maximum to its right. However, that would be O(n²). I realized that if I start from the **end of the array** (looking in the solutions), I can keep track of the maximum value seen so far in a single pass.

> Key Insight: Iterating backwards allows us to know the "greatest element to the right" instantly because we've already processed those elements.

---

## 🚀 Final Solution

    class Solution {
        public int[] replaceElements(int[] arr) {
            int max = -1;
            
            // Iterating from right to left
            for (int i = arr.length - 1; i >= 0; i--) {
                int currentVal = arr[i];
                arr[i] = max; // Replace current with the max found to its right
                
                // Update max for the next element to the left
                if (currentVal > max) {
                    max = currentVal;
                }
            }
            return arr;
        }
    }

---

## 🛠️ Algorithm
1. Initialize Max: Start with `max = -1` (as required for the last element).
2. Backward Loop: Start from `arr.length - 1` down to `0`.
3. Temporary Store: Keep the current element's value before overwriting it.
4. Replace: Set `arr[i]` to the current `max`.
5. Update Max: Compare the stored current value with `max`. If it's larger, update `max`.
6. Return: The array is modified in-place.

---

## 📊 Complexity Analysis
* Time Complexity: O(n) — We traverse the array exactly once.
* Space Complexity: O(1) — We modify the array in-place and only use one extra variable (`max`).

## 📝 Code Notes
* In-place modification: This is highly efficient as it doesn't require allocating a new array.
* The variable `currentVal` (or `temp`) is crucial because once you do `arr[i] = max`, you lose the original value needed to update the `max` for the next iteration.
