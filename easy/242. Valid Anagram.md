# 📝 Valid Anagram

| Difficulty | Topics |
| :--- | :--- |
| Easy | String, Sorting, Hash Table |

## 📖 Problem Description
Given two strings `s` and `t`, return `true` if `t` is an anagram of `s`, and `false` otherwise.

An **Anagram** is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

---

## 💡 Initial Thoughts & Evolution
My first instinct was to use a `HashMap` to count character frequencies. While it works and has $O(n)$ time complexity, the overhead of object creation (Autoboxing `int` to `Integer`) and hash collisions made it feel like overkill for this specific problem.

> Key Insight: If two strings are anagrams, their sorted versions must be identical.

### ❌ Initial Attempt (HashMap Approach)
    class Solution {
        public boolean isAnagram(String s, String t) {

            //Maybe, a hashMap can be overkill sometimes. I knew that looping twice was too much :/
            char[] arrayS = s.toCharArray();
            char[] arrayT = t.toCharArray();

            Arrays.sort(arrayS);
            Arrays.sort(arrayT);

            return Arrays.equals(arrayS, arrayT);
        }
    }
---

## 🚀 Final Solution
By converting the strings to character arrays and sorting them, we can directly compare the results.

    class Solution {
        public boolean isAnagram(String s, String t) {
            // Early exit if lengths differ
            if (s.length() != t.length()) return false;

            char[] arrayS = s.toCharArray();
            char[] arrayT = t.toCharArray();

            Arrays.sort(arrayS);
            Arrays.sort(arrayT);

            return Arrays.equals(arrayS, arrayT);
        }
    }

---

## 🛠️ Algorithm
1. Length Check: If lengths are different, they cannot be anagrams.
2. Conversion: Convert both strings to `char[]` arrays.
3. Sorting: Use `Arrays.sort()` (Dual-Pivot Quicksort) to put characters in order.
4. Comparison: Use `Arrays.equals()` to check if both sorted arrays match.

---

## 📊 Complexity Analysis
* Time Complexity: O(n log n) — Due to the sorting step.
* Space Complexity: O(n) — To store the character arrays.

## 📝 Code Notes
* Trade-off: While $O(n \log n)$ is technically slower than $O(n)$, sorting primitives is extremely fast in Java and often beats HashMaps for smaller constraints.
* Optimization: For an $O(n)$ solution with $O(1)$ space, you could use an `int[26]` array as a frequency counter (since the problem usually specifies lowercase English letters).
