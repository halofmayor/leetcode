# 📝 Is Subsequence

| Difficulty | Topics |
| :--- | :--- |
| Easy | Two Pointers, String |

## 📖 Problem Description
Given two strings `s` and `t`, return `true` if `s` is a subsequence of `t`, or `false` otherwise.

A **subsequence** is formed by deleting some (or no) characters from the original string without disturbing the relative positions of the remaining characters. (e.g., "ace" is a subsequence of "abcde" while "aec" is not).

---

## 💡 Initial Thoughts & Evolution
The core rule is "relative position." This means I don't need to find all characters at once; I just need to find each character of `s` inside `t` in the correct order. The most efficient way is to iterate through the main string and maintain a counter to track my progress in the subsequence.

---

## 🚀 Final Solution

    class Solution {
        public boolean isSubsequence(String s, String t) {
            // Edge case: an empty string is always a subsequence
            if (s.length() == 0) {
                return true;
            }

            char[] sub = s.toCharArray();
            int count = 0;

            for (int i = 0; i < t.length(); i++) {
                // Check if current character in 't' matches the next needed char in 'sub'
                if (sub[count] == t.charAt(i)) {
                    // If we matched the last character of 's', we are done
                    if (count == s.length() - 1) {
                        return true;
                    } 
                    // Move to the next character in the subsequence
                    count++;
                }
            }
            return false;
        }
    }

---

## 🛠️ Algorithm
1. Empty Check: Immediately return `true` if `s` is empty.
2. Conversion: Convert string `s` to a character array `sub` for potentially faster access.
3. Tracking: Use a variable `count` to keep track of how many characters of the subsequence have been matched.
4. Iteration: Loop through the target string `t`.
5. Match & Exit: Each time a character matches `sub[count]`, check if it was the last one needed (`count == s.length() - 1`). If so, return `true`. Otherwise, increment `count`.
6. Final Result: If the loop finishes without matching all characters, return `false`.

---

## 📊 Complexity Analysis
* Time Complexity: O(n) — Where n is the length of the string `t`. In the worst case, we check every character of `t` once.
* Space Complexity: O(m) — Where m is the length of string `s`, because `s.toCharArray()` creates a new array of size m.

## 📝 Code Notes
* Performance: By converting `s` to `toCharArray()`, you avoid repeated `charAt()` calls on string `s` inside the loop.
* Logic: The `count` variable acts as a pointer that only moves forward when a match is found, preserving the relative order.
