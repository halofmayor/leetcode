# 📝 Number of Senior Citizens

| Difficulty | Topics |
| :--- | :--- |
| Easy | Array, String, Simulation |

## 📖 Problem Description
You are given a 0-indexed array of strings `details`. Each string has a fixed length of 15 and contains compressed passenger information:
- Characters 0-9: Phone number.
- Character 10: Gender.
- **Characters 11-12: Age.**
- Characters 13-14: Seat number.

Return the number of passengers who are **strictly more than 60 years old**.

---

## 💡 Initial Thoughts & Evolution
The problem is a straightforward parsing task. Since the age is always at the same position (indices 11 and 12), we don't need to split or regex the string. We just need to extract those two characters, convert them to an integer, and compare them.

> Key Insight: Manual character-to-integer conversion (`char - '0'`) is faster and uses less memory than `substring()` combined with `Integer.parseInt()`.

---

## 🚀 Final Solution

    class Solution {
        public int countSeniors(String[] details) {
            int count = 0;
            for (int i = 0; i < details.length; i++) {
                String current = details[i];
                
                // Extracting age from fixed positions 11 and 12
                // Index 11 is the tens place, Index 12 is the units place
                int age = (10 * (current.charAt(11) - '0')) 
                                + (current.charAt(12) - '0');
                
                if (age > 60) {
                    count++;
                }
            }
            return count;
        }
    }

---

## 🛠️ Algorithm
1. Counter: Initialize a `count` variable at 0.
2. Iteration: Loop through each string in the `details` array.
3. Math Conversion: 
    - Take the character at index 11, subtract the ASCII value of '0' to get the integer, and multiply by 10.
    - Take the character at index 12, subtract '0', and add it to the previous result.
4. Conditional: If the calculated `age` is greater than 60, increment the counter.
5. Result: Return the final count after the loop finishes.

---

## 📊 Complexity Analysis
* Time Complexity: O(n) — We visit each string in the array exactly once.
* Space Complexity: O(1) — We only use a constant amount of extra space (variables `count` and `age`).

## 📝 Code Notes
* ASCII Trick: Subtracting `'0'` from a numeric `char` is the standard way to get its literal integer value without overhead.
* Fixed Indexing: The problem guarantees the strings are length 15, so `charAt(11)` and `charAt(12)` are always safe.
