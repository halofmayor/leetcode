# 📝 Concatenation of Array

| Difficulty | Topics |
| :--- | :--- |
| Easy | Array, Simulation |

## 📖 Problem Description
Given an integer array `nums` of length `n`, you want to create an array `ans` of length `2n` where `ans[i] == nums[i]` and `ans[i + n] == nums[i]` for `0 <= i < n`.

Essentially, `ans` is the concatenation of two `nums` arrays.

---

## 💡 Initial Thoughts & Evolution
The problem is straightforward: we need to duplicate the array content. I realized that instead of looping twice (once for the first half and once for the second), I could just fill both slots (`i` and `i + n`) in a single pass. 

> Key Insight: Since the second half of the array starts exactly at index `n`, we can map the current element to both positions simultaneously.

---

## 🚀 Final Solution

    class Solution {
        public int[] getConcatenation(int[] nums) {
            int n = nums.length;
            int[] ans = new int[2 * n];
            
            for (int i = 0; i < n; i++) {
                ans[i] = nums[i];          // Fill first half
                ans[i + n] = nums[i];      // Fill second half
            }
            
            return ans;
        }
    }

---

## 🛠️ Algorithm
1. Get Length: Store the length of `nums` as `n`.
2. Initialize: Create a new array `ans` with size `2 * n`.
3. Single Pass: Iterate from `0` to `n - 1`.
4. Double Assignment: 
    - Set `ans[i]` to `nums[i]`.
    - Set `ans[i + n]` to `nums[i]`.
5. Return: Return the completed `ans` array.

---

## 📊 Complexity Analysis
* Time Complexity: O(n) — We traverse the input array exactly once.
* Space Complexity: O(n) — We create a new array of size 2n to store the result.

## 📝 Code Notes
* Using `i + n` is a very efficient way to "offset" your position into the second half of the array without needing a nested loop or a separate counter.
